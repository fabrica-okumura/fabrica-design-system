# 同期リリース手順

## トリガー
- Aリポ変更をBリポへ反映したい相談。
- リリース・引き渡し準備。
- 同期対象/非対象の確認依頼。

## 手順
1. 変更ファイルを洗い出し、次の2区分に分類する。
   - 同期対象（再利用資産）
   - A専用docs資産
2. A専用資産が同期期待に含まれていないことを確認する。
3. 境界ルールの前提が変わったら README を更新する。
4. 回答には短い同期チェックリストを添える。
   - Bで同期コマンドを実行
   - 依存差分を確認
   - 同期対象パスに意図しない編集がないか確認

## ガードレール
- A専用docs変更をB同期変更として説明しない。
- チェックリストは短く、実行可能な形にする。
