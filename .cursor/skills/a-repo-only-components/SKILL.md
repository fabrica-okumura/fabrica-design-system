# Aリポ専用コンポーネント

## トリガー
- 「Aリポだけで使うコンポーネント」を追加するとき。
- docsサイト専用のUIやレイアウト部品を作るとき。
- Bリポへ共有するコンポーネントを流用すべきか迷ったとき。

## 手順
1. 要件がAリポ専用かを確認する。
2. Aリポ専用であれば、Bリポ同期対象のコンポーネントは使用しない。
3. Aリポ専用コンポーネントを `src/components/docs-site/**` に新規作成する。
4. 利用箇所は `src/app/**` から参照し、再利用資産ディレクトリへ混在させない。
5. 必要に応じて README の責務分離ルールと整合しているか確認する。

## ガードレール
- Aリポ専用要件で `src/components/ui/**` や `src/components/layout/**` の部品を流用しない。
- Aリポ専用コンポーネントを Bリポ同期対象として扱わない。
